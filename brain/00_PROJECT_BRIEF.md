---

## Security non-negotiables
- `service_role` key: never in browser or committed to GitHub
- RLS enabled on ALL tables — no exceptions
- Users can only read/write their own rows (`auth.uid() = user_id`)
- `.env` is gitignored — never commit it
- Pre-commit hook scans for leaked secrets
- Config served via build step (`config.js` generated from `.env`)

## Conventions
- American English in all UI text
- Mobile-first CSS, no inline styles — CSS variables only
- Sentence case on all labels and headings
- Files: kebab-case | CSS vars: --kebab-case | JS: camelCase | DB: snake_case
- Git branches: feat/name · fix/name · chore/name
- Commit messages: feat: / fix: / chore: / docs: prefix
- Migrations: append-only — never edit existing files, always create new
